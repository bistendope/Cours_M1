  32 /*
  33  * The VFS-interface operations that can write to the fs are:
  34  *
  35  *    sync
  36  *    fsync
  37  *    write
  38  *    reclaim
  39  *    truncate
  40  *    creat
  41  *    symlink (not supported by default and which we are going to ignore)
  42  *    mkdir
  43  *    link
  44  *    remove
  45  *    rmdir
  46  *    rename
  47  *
  48  * The sync operations are not interesting by themselves. Also,
  49  * because the VFS-level unmount operation works by first syncing and
  50  * then doing an unmount (which by default at least asserts nothing
  51  * further needs to be written out) there is nothing to be gained by
  52  * combining sync with write operations and seeing if the behavior is
  53  * different: if sync doesn't produce a clean fs, unmount won't
  54  * either, and vice versa.
  55  *
  56  * This means there are basically the following families of cases to
  57  * consider:
  58  *    - writing to files in various ways and patterns
  59  *    - truncating files
  60  *    - directory ops that create things
  61  *    - directory ops that remove things (various combinations with reclaim)
  62  *    - link and rename
  63  */
